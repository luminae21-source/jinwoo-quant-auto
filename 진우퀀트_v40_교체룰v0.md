# 진우퀀트 v4.0 — 교체룰 v0 (Phase 1 단독)

> 작성일: 2026-05-24
> 적용 범위: Phase 1 (CAPM 시장 β 분해) 출력만으로 평가 가능한 룰
> 다음 단계: Phase 2 (산업 ETF) 추가 시 T2 정확도 향상 예정

---

## 0. 룰 설계 원칙

진우퀀트 영역 4의 핵심 질문은 **"왜 빠졌는지 모르면 팔지 말지 판단 못 한다"**. Phase 1은 시장 동조와 종목 고유를 1차 분해해서 그 질문에 답할 수 있는 가장 단순한 도구다.

설계 원칙:

1. **단순함 우선.** Phase 1만으로 평가 가능한 룰만. 산업·펀더멘털 신호는 Phase 2~3로 미룸.
2. **False signal 방지.** 단일일 변동은 잡음 가능성 높음. 트리거가 발동해도 즉시 매매 X. 분류만.
3. **체력 점수와 분리.** 교체룰은 체력 점수 변화와는 별개 신호. 두 신호가 겹칠 때 강한 액션, 한쪽만 발동 시 관찰.
4. **진우 라이프스타일.** 자동 매매 아님. 트리거가 모바일 대시보드에 표시되면 그 주말에 진우가 검토 → 의사결정.

## 1. Phase 1 분해 정의

매일 종목 i의 수익률을 다음과 같이 분해:

```
r_i,t = α_i + β_i × r_KOSPI,t + ε_i,t
```

여기서 β_i는 60영업일 rolling, 1일 lag (look-ahead 방지).

- **market_contrib_t** = β_{t-1} × r_KOSPI,t
- **idio_t**            = r_i,t − market_contrib_t  (= α_i + ε_i,t)
- **idio_z_t**          = (idio_t − μ_60) / σ_60     (60일 rolling z)

## 2. 트리거 (T1 ~ T5)

### T1 — 단일일 이벤트 의심 [severity: 🔴 high]

조건: 최근 N영업일 내 |idio_z| ≥ 3.0인 날이 있음 (기본 N=20, z=3.0)

해석: ε이 60일 표준편차의 3배를 넘는 충격. 통상적 잡음으로 설명 불가. 거의 항상 이벤트 (실적·공시·뉴스·매크로 충격) 동반.

액션:
- 해당 날짜 ±1영업일 뉴스·공시(DART) 확인
- 펀더멘털 관련 (실적 미스/서프라이즈) → 다음 분기 F-Score 재평가 예고
- 단순 시장/소문 (M&A 루머·외인 청산 등) → 1주일 reversal 여부 관찰

---

### T2 — 종목 고유 약세 누적 [severity: 🔴 high]

조건: 최근 20영업일 Σ idio ≤ −10%

해석: 시장 빼고도 종목만으로 -10% 누적. KOSPI가 보합·상승해도 종목만 부진 = 펀더멘털 의심.

액션:
- F-Score 재확인, 분기 결산 우선 체크
- **2개월 연속 T2 발동** 시 비중 50% 축소
- 3개월 연속 시 universe 교체 후보

---

### T3 — 시장/산업 동조 약세 [severity: 🟢 low — 보유 유지 신호]

조건: 최근 20영업일 Σ r_stock ≤ −10% **AND** Σ idio > −3%

해석: 종목이 빠졌지만 시장 동조. 종목 본연의 문제 아님. 매도 시 헛수고.

액션:
- **보유 유지.** "왜 빠졌나" 답이 이미 나옴 (KOSPI 따라간 거).
- v3.6 매매룰의 trailing stop 알림이 떠도 T3 발동 시 무시 가능.
- 시장 회복 시 정상 회귀 기대.

> 비고: Phase 2에서 산업 ETF factor 추가되면 T3가 더 정밀해짐. 현재는 "시장 vs 종목"만이라 "동일 산업 종목 동반 약세"가 idio에 섞임.

---

### T4 — 종목 고유 호조 [severity: ℹ️ info]

조건: 최근 20영업일 Σ idio ≥ +10%

해석: 시장보다 종목 자체가 강함. 알파 발생 중.

액션:
- Mom12 점수와 교차 확인 — Mom12도 +면 추세 강화 신호 (FAR_BUY 4분할 가속 검토)
- Mom12 0 또는 음수인데 T4 발동 = 단발성 호재 가능성 (지속력 의심)
- 비중 추가 매수는 v3.6 매매룰 (Kelly sizing) 범위 내에서만

---

### T5 — β 급변 [severity: 🟡 med]

조건: 현재 β_60d − 30일 전 β_60d 의 절대값 ≥ 0.4

해석: 종목 성격 변화 (예: 안정주 0.7 → 변동주 1.2). 산업 사이클·기업 구조 변화 등 trigger.

액션:
- Kelly sizing 재계산 — β 올랐으면 비중 상한 ↓, 내렸으면 ↑
- v3.7 BAB 점수 다음 갱신 시 자연스럽게 반영됨 (BAB가 β 기반이라)
- 30일 더 관찰 후 새 β가 안정되는지 확인

## 3. 트리거 조합 케이스

여러 트리거가 동시 발동 시:

| 조합 | 의미 | 액션 |
|---|---|---|
| T1 + T2 | 이벤트 + 약세 누적 (이벤트 후 회복 못 함) | **즉시 정밀조사**, 비중 축소 검토 |
| T2 + T5 | 약세 + β 상승 | risk-off로 가는 종목. 비중 축소 우선 |
| T3 + (T1 없음) | 시장 약세 + 이벤트 없음 | 보유 유지 (가장 안전) |
| T4 + Mom12+2 | 알파 + 모멘텀 강화 | 추가 매수 검토 (단, Kelly 상한 준수) |
| T4 + Mom12 음수 | 알파 있지만 추세 약함 | 관망 |

## 4. v3.6 매매룰과의 관계

v3.6 매매룰은 가격 기반 (trailing stop -8%/-15%/-25%). 교체룰 v0는 **수익률 분해 기반**으로 trailing stop의 노이즈를 거른다.

| v3.6 신호 | Phase 1 교차 확인 | 액션 |
|---|---|---|
| Trailing stop -8% 알림 | T3 발동 (시장 동조) | 무시 — 시장 따라간 것 |
| Trailing stop -15% 비중 축소 | T2 발동 (종목 약세) | 그대로 실행 — 진짜 약세 |
| Trailing stop -15% 비중 축소 | T3 발동 + T2 없음 | **보류** — 시장 회복 기다림 |
| Trailing stop -25% 강제 청산 | T1+T2 발동 | 그대로 실행 |
| 가격 변동 없는데 등급 2단계 하향 | (가격 신호 없음) | v3.6 룰대로 매도 |

## 5. 한계 (Phase 2~4로 미룸)

### Phase 1 단독의 약점

1. **산업 동조 = 종목 고유로 잘못 분류** — 카카오 -5%인데 KOSPI는 +1%, NAVER도 -4%라면 "인터넷 산업 동조"인데 Phase 1은 "카카오 종목 고유 약세"로 본다. Phase 2 (산업 ETF factor) 도입 시 해결.

2. **펀더멘털 변화 detection 불가** — 분기 실적이 나빠진 종목과 단순 시장 노이즈 구분 불가. Phase 3 (F·Mom·Quality factor 회귀) 필요.

3. **이벤트 자동 분류 불가** — T1이 발동하면 "이벤트다"까지는 알지만, "M&A인지 실적 미스인지" 분류 못 함. Phase 4 (event detection — 뉴스/공시 NLP) 필요.

### 18종목 통계적 한계

- 일 단위 분해는 N=1 (각 종목 자기 자신과의 시계열)이라 통계적으로 robust. 18종목 cross-sectional 분석은 의미 없음.
- 따라서 Phase 1은 종목별 독립 분석이고 portfolio attribution이 아님.
- Portfolio attribution은 비중 데이터 (현재는 동일가중 가정)와 결합해서 별도 모듈 필요.

## 6. 운용 가이드

### 매일 (자동)
1. `attribution_v40_phase1.py` 실행 (GitHub Actions 또는 진우님 PC scheduler)
2. `attribution_panel.html` 갱신
3. JSON 결과 저장 (이력 누적)

### 주말 (진우님 검토)
1. 패널에서 발동 trigger 확인
2. T1·T2 발동 종목 — 뉴스/공시 조사 (DART, 네이버 금융)
3. T3 발동 종목 — 안심하고 보유
4. T4 발동 종목 — Mom12와 교차 확인 후 sizing 결정
5. 모든 트리거 결과를 매매 로그에 기록 → out-of-sample 검증 데이터로 누적

### 월말 (효과 검증)
1. 이전 달 트리거가 실제 결과와 맞았나? (예: T2 발동 종목이 다음 달 정말 부진했나?)
2. False positive rate 추정 → z 임계값 또는 누적 % 컷오프 조정 검토
3. 1년 누적되면 본격 calibration 가능
