# 진우퀀트 실전 운영 가이드 — 가치·피크 오버레이 통합 (v1.0)

> 작성일: 2026-06-07 · 대상: 진우퀀트 일상 운영 · ⚠️ 투자자문 아님, 매매는 본인 판단·책임
> 전체 분석 근거는 `진우퀀트_가치투자_피드백.md`(§1~13) 참조. 이 문서는 그걸 *운영용*으로 압축.

## 0. 한눈에 (TL;DR)

- **엔진** = 퀄리티 + 모멘텀. PIT 검증에서 이 둘만 안정적 초과수익(IR +0.8~1.0). 밸류 단독은 중립, 퀄리티 단독은 마이너스.
- **방어** = MA200(추세) + 퀄리티 필터. 밸류를 방어 슬리브로 섞는 건 비효율(MDD 1~3%p↓ 위해 CAGR 5~10%p↓).
- **밸류·피크** = 매일 뜨는 **경보(트림권고)**. 기계적 자동 트림은 v40 합격선 미달 → **사람이 보고 판단**.
- **지금 최우선** = 반도체 비중·트림 판단(SK하이닉스·한미반도체가 비쌈+과열 경보, MA200 대비 +90~118% 과열).
- **현실 기대** = 시장 +5~10%p(연 17~25%). "76%"는 사후편향, 쓰지 말 것.

## 1. 왜 이렇게 운영하나 (검증된 결론)

PIT(그때 정보만으로) 백테스트로 광범위 유니버스 사전선택을 검증한 결과(2021–25, 2026 급등 제외해도 유지):

| 팩터 슬리브 | 순수효과(IR_EW) | 판정 |
|---|---|---|
| Momentum | **+0.76** | 엔진 |
| Q→Mom (퀄리티 필터 + 모멘텀) | **+0.83 (최고)** | 엔진 |
| Value(밸류) 단독 | +0.08 | 중립 → 선택도구 |
| Quality(퀄리티) 단독 | −0.21 | 필터로만 |

세 줄 요약:
1. **알파는 "어떤 종목을 골라 오래 들었나"에 있다** — fixed18(회전0%)이 IR 1.34로 모든 룰을 이김.
2. **밸류의 진짜 쓸모는 수익이 아니라 "선택·트림"** — 비싸고 과열된 종목을 거르는 데 쓴다(방어 슬리브 X).
3. **기계적 피크-트림은 백테스트 미달** — 2021–26엔 비싼+과열=승자(반도체)였으니까. 그래서 **경보로만** 쓰고 실행은 사람이 결정.

## 3. 운영 루틴

### 매일 (자동 — 손 안 댐)
- GitHub Actions가 16:30 KST에 점수 산출 → 오버레이 → `docs/overlay_scores.csv`에 트림권고 게시.
- 폰에서 대시보드(`https://luminae21-source.github.io/jinwoo-quant-auto/`) 확인. **TRIM 경보 뜬 종목만** 눈여겨보면 됨.

### 매월 (수동 ~10분)
1. `python ma200_regime_check.py` → 방어상태. RISK-OFF 종목 = 추세 꺾임(룰상 관망).
2. `python holdings_trim_analysis.py` → 보유 스냅샷(퀄/모멘/밸류 백분위) + 방어블렌드.
3. **트림권고(TRIM) 종목을 §4 규칙으로 판단** → 트림 여부 결정.
4. (트림한다면) `python trim_simulation.py`로 비중·스트레스 영향 확인.

### 분기 (수동)
- `python value_quality_backtest.py` → 모멘텀·Q→Mom 우위가 아직 유효한지(레짐 변하면 결론도 변함).
- DART 재무 갱신 → `v37_2` 점수 재계산(기존 분기 루틴).

## 4. 의사결정 규칙 (사람이 판단)

### 4-1. 트림 경보 읽는 법
- **TRIM(비쌈+과열)** = B/M 하위(비쌈) **AND** MA200 대비 과열(반도체 +25%·경기민감 +30%·기본 +40% 초과). → **차익실현 1순위 후보.**
- **WATCH(비쌈)** = 비싸지만 과열 임박. → 관찰.
- **순위Δ 하락(−2 이상)** = 오버레이가 강등시킨 종목. → 비중 점검.
- ⚠️ 경보 = 자동 매도 아님. "왜 비싼가? 정상이익인가 피크이익인가?"를 본인이 답한 뒤 결정.

### 4-2. 트림하면 어디로? (§11 검증)
- **다른 주식 재투자 ✗** — 조정장엔 다 같이 빠져 분산 안 됨(MDD 오히려 악화).
- **현금/방어 ○** — 트림분을 현금으로 들면 변동성·MDD 실제 개선(예: Exit→현금 변동성 27.7→19.0%, MDD −20.1→−16.6%).
- **트림폭 = 방어폭** — 반도체 −40% 조정 시: Baseline −10.0% vs Trim½ −5.8% vs Exit −1.7%.

### 4-3. 밸류 기회 (역방향 — 고퀄·저모멘텀)
- 점수는 낮지만(모멘텀 약) 사업이 좋은 종목은 *밸류기회*일 수 있음. **2분 스토리** 못 쓰면 안 산다.
- 예: **아모레퍼시픽** = 진행형 턴어라운드(매출 바닥 찍고 회복, 북미 +35%) → 밸류기회 쪽. **삼양식품** = 초우량 고성장(ROE 30%·마진 22%)인데 비싸서 조정 → 트랩 아님, *합리적 가격까지 대기*.

## 6. 정직성 규율 (멍거 체크리스트)

- ☑ 백테스트 통과 못 한 룰은 **자동 채택하지 않는다**(피크-트림 = 경보로만).
- ☑ forward 기대 = **시장 +5~10%p**. backtest 41/76%·월간36% 같은 숫자를 기대치로 쓰지 않는다.
- ☑ 결론은 **PIT 숫자**(보수적)로 잡는다. fixed18·생존편향 시계열의 화려한 수치를 믿지 않는다.
- ☑ 새 룰 채택 전 `backtest_v40_regime.py` 합격선(dMDD≥+2%p AND dCAGR≥−1%p AND Sharpe·IR 비열위)으로 검증.

## 7. 캐비엇·한계

- **생존편향**이 모멘텀을 가장 부풀린다(가격파일=현 구성종목). 실제 forward 모멘텀 엣지는 측정치보다 작다.
- **2026 후반 데이터**(SK하이닉스 월 +81% 등)는 검증 불가 극단치 → 절대수익은 신뢰 보류(제외해도 결론은 유지).
- **단일 시장·단일 시대.** 모멘텀 크래시(2009·2016형)는 표본에 없음.
- 본 시스템은 의사결정 *지원* 도구. 자동매매 아님. 매매는 본인 판단·책임.
