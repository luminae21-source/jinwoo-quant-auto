## 4. 집행 제약 — 소액 계좌 빌드 룰 (가격 실측 기반)

종목당 목표 7.69%, 종목 cap 15% 기준. **1주 가격이 트랙 평가액 T의 15%를 넘으면 그 종목은 아직 못 산다**: 편입 가능 조건 = 주가 ≤ 0.15 × T.

| 편입 단계 | 필요 트랙 규모 T | 종목 (6월 근사가) |
|---|---|---|
| 즉시 (~250만) | 21만~190만 | NH투자증권(3.1만)·두산에너빌리티(10만)·아모레퍼시픽(10.9만)·KT&G(18.5만)·ISC(21.2만)·한미반도체(27.6만)·NAVER(28.1만) |
| T ≥ ~240만 | 238만·241만 | 알테오젠(35.7만)·삼성전자(36.1만) |
| T ≥ ~330만 | 324만 | 삼성물산(48.6만) |
| T ≥ ~720만 | 713만·760만 | 한화에어로(107만)·삼양식품(114만) — 각 1주부터 |
| T ≥ ~1,570만 | 1,573만 | SK하이닉스(236만) 1주 — 그 전까지 **미보유 허용** |

**트랜치 당일 절차**: ① 당일 S+/S/A 리스트 확정 → ② T(트랙 평가액+신규 현금) 계산 → ③ 편입 가능 종목 n개 선별(등급·순위순) → ④ 종목당 목표 = T/n, 수량 = 목표÷주가 반올림(최소 1주) → ⑤ 매수.

워크드 예시 (실측 기준 — 머티 500만, 적립 50만/월):
- T1: 재원 ≈ 167만+50만 = 217만 → 편입 가능 7종목(주가 ≤ 32.5만), 종목당 ~31만
- T2: T ≈ 434만 → 알테오젠·삼성전자·삼성물산 추가 (10종목, 종목당 ~43만)
- T3: T ≈ 651만 → 12·13번째(한화에어로·삼양식품)는 아직 미달 → 9~10월 적립으로 T≥720만 도달 시 1주씩
- 가격은 6월 진행분 근사치 — 실행 시 실제 호가 기준으로 ②~④ 재계산.

### 9-1. dashboard 멈춤 진단 (2026-06-06 정정 — 원인 재확인 완료)
- **정정**: 앞서 의심한 "Node24 push 대기"는 **오진**. origin/main에 daily.yml Node24 범프(checkout@v5·setup-python@v6·configure-pages@v5)가 **이미 push돼 있음**(b7731c2까지 반영 확인). 미push 커밋은 본 B전환 문서 2건(마크다운, 워크플로 무관)뿐.
- **유력 원인 = Validation Gate 롤백**: daily.yml은 score 생성 실패/검증 실패 시 deploy를 건너뛰고 **직전 dashboard를 그대로 유지**(설계된 안전동작). dashboard가 특정일(6/2)에 정확히 고정된 것이 이 패턴과 일치 — 단순 stale이 아니라 **6/3부터 매 실행이 hard-fail해 롤백 중**일 개연성.
- **hard-fail 트리거 후보**(validate.py issues, deploy 차단): ① 신선도 — score_v37_2.py가 CSV 재생성 실패(DART/FDR 페치 에러) 시 직전 CSV >24h → FAIL ② NaN/inf 체력_최종 ③ 18종목 누락 ④ 합불일치 ⑤ 비중합≠100%. (※ 6월 급락의 "급변동"은 **경고**라 deploy 차단 안 함 — 급락 자체는 원인 아님.)
- **PC 점검 순서** (샌드박스에선 Actions 로그 접근 불가 → 진우님 직접):
  1. GitHub repo → **Actions 탭** → 최근 run 빨강(❌) 확인 → 실패한 step 이름(보통 score_v37_2 또는 Validation Gate)

  3. 페치 일시 에러면 **workflow_dispatch 수동 1회 실행**으로 복구 시도 / 데이터 결함(NaN·종목)이면 다음 세션에서 코드 점검
- **트랜치는 dashboard 없이도 가능**: 멈춰 있어도 로컬에서 `python score_v37_2.py` 1회 실행 → 콘솔/`dashboard_v37_2.html`로 당일 S+/S/A 등급 확보 후 T1 집행 가능(운영가이드 §2 백업 경로).
