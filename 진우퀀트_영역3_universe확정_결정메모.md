## 1. 문제 (06-03 §7-3, 진우 인사이트)

KOSPI/KOSDAQ를 pool해 단일 랭크 + 전역 사이즈·유동성 필터를 걸면
(a) 작은 KOSDAQ가 통째로 밀리고 (b) 비교 불가한 사이즈 티어가 섞임.
실측: 알테오젠 pool r356 → KOSDAQ-상대 r99/229, ISC r290 → r86/229 (교차시장 왜곡 확인).

## 2. 구현 (universe_market_relative.py, selftest 통과)

1. **시장별 분리 스크리닝** — `PB.score_at`을 시장별 부분집합으로 실행 (가격·자기시장 EW·시장 내 분위수 전부 분리 = kosdaq_relative_screen이 예고한 '다음 단계')
2. **KOSDAQ carve-in** — KOSDAQ 내 시총 top-50 ∩ ADTV 상위 50%만 후보
3. **결합** — universe 30 = KOSPI 룰셋 (30−q) + carve-in KOSDAQ q
- market_map.csv는 liquidity 파일 기반 자동 생성 (FDR 불필요 → 샌드박스 실행 가능, 2,770종목: KOSPI 948 / KOSDAQ 1,822)

## 5. 정직성 한계 (사전 기록)

1. **백테스트는 점수 + 정적 시장라벨만 사용** — 시총·ADTV 가드레일은 '현재' 스냅샷이라 과거에 적용하면 look-ahead. 가드레일은 운용(제안) 단계 전용.
2. **시장라벨도 현재 스냅샷** — 상장폐지·이전상장 미반영, survivorship bias 잔존 (기존 sweep·PIT 백테스트와 동일 한계).
3. **KOSDAQ 섹터 cap은 산업이 아니라 시장부(우량기업부 등) 기준** — liquidity_kosdaq.csv에 산업분류 없음. 원익IPS·이오테크닉스·하나마이크론은 전부 반도체 장비/소재 계열 = 산업 집중 위험 미차단 → **사람 검토로 보완 + PC에서 산업분류 보강 (영역 3 모듈에서)**.
4. 인샘플 + 비용 무차감 — 비용·합격선 재설계는 영역 3 모듈 ③의 몫 (코스닥 왕복 0.5~0.7% prior 등록됨).
5. 2026-06은 미완성 월 stub (D 포렌식과 동일) — 동시점 병행 비교라 두 팔에 동일 적용, 판정 유효.

### 해석 (정직)
1. KOSDAQ 슬롯은 6회 리밸 전부 충원(2/2, 4/4)됐는데도 IR이 낮다 = **carve-in 자체가 drag** (슬롯 미충원 문제 아님). 시장-상대 분리는 분류 정합성은 옳지만, 예약 슬롯이 pool 기준 약한 종목을 강제 편입하는 비용이 더 컸다.
2. 사전 등록 기각 누적 7건째 (GP·AG·EarnMom·Value·regime익스포저·regime가중치·MR carve-in) — "그럴듯한 아이디어"가 게이트에서 걸러지는 시스템이 정상 작동.
3. pooled_top30의 시장초과 +0.8%p/IR 0.07은 사실상 0 — D §6과 일치 (광범위 벤치마크 대비 체계적 엣지 작음). **이 백테스트의 역할은 universe 구성 방식(pooled vs MR) 판정까지.** 확장 universe의 수익성 판정은 영역 3 모듈에서 v3.7.2 공식 엔진 + 비용 재설계로 별도 수행.
4. 06-03 sweep(top-30: 24.1%/IR 0.12)과 근사 재현 — 차이는 2026-06 급락 stub 반영 여부 (D의 참조 staleness 교훈 그대로. 동시점 병행 비교라 판정 유효).
