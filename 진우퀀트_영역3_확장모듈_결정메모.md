## 2. 사전 등록 ① — 점수 정의 (2026-06-05 진우 채택: **PIT-proxy**)

`pit_universe_backtest.score_at`과 동일 정의, 공식 엔진에서는 월별 산출:

| 컴포넌트 | 정의 | 범위 |
|---|---|---|
| 체력 | F(정수 Piotroski 0~9, FY−1) × 12/9 | 0~12 |
| Sloan | accrual = (NI−CFO)/assets, 5분위 (낮을수록 +) | +2~−2 |
| NOA | noa_ratio 5분위 (낮을수록 +) | +2~−2 |
| Mom12 | production 동일 구간 매핑 (`_mom_score`) | +2~−2 |
| BAB | β60 production 동일 구간 매핑 (`_bab_score`) | +2~−2 |
| Echo | t−12~t−7 수익률 rank ≥0.8 → +1 / ≤0.2 → −1 | +1~−1 |

- 등급·선별 = production 컨벤션 그대로: `grade()` 절대 컷, picks = {S+, S, A}(점수 ≥9), EW + caps(종목 15%·섹터 35%), picks 0인 달은 현금(0%). **컷 튜닝 금지.**
- 채택 사유: F_korean 연속값(6.21 등)은 산출 스크립트가 저장소에 없는 Colab 수작업 유산 → 복제 시 정의 드리프트 위험. PIT-proxy는 universe 선정·검증과 동일 정의 + fundamentals_pit로 완전 자동화. ModF·FAR 생략(보조 항 전례, PIT검증 설계 §5).
- **명시: PIT-proxy ≠ production 점수.** univ30은 별도 트랙 — 18종 production 점수표와 직접 비교 금지.

## 5. 사전 등록 ④ — 합격선 (승인 후 변경 금지)

**A. universe 게이트** (2026-06-05 진우 선택: **시장 상대**) — 같은 실행의 KOSPI 라인 대비, 셋 다 충족 시 PASS:

1. CAGR ≥ KOSPI +3.0%p
2. IR ≥ 0.30
3. MDD ≤ KOSPI (시장보다 비악화)

FAIL → 모듈 종결. 룰 universe는 분기 재스크린 정보 트랙으로만 유지(운용 미편입), 재시험·재튜닝 금지.

**B. regime 게이트** (D §2 재사용, 진우 확정) — **univ30 base 대비**, 셋 다 충족 시 PASS:

1. MDD ≥ +2.0%p 개선
2. CAGR ≥ −1.0%p
3. Sharpe·IR ≥ base −0.01

변형 2개(regime_w/s)만, `REGIME_MULTS`·hysteresis·detector 임계값 무수정, **재시험 1회 한정**. 둘 다 FAIL → regime 종결(18종 기각과 합산 기록). regime 채택은 A PASS 전제 — A FAIL 시에도 산출은 기록(정보용).

**C. PASS 시**: production 즉시 교체 없음 — C 패턴 **병행 관찰 모드**부터.

## 10. 정직한 prior·한계 (사전 기록)

1. pooled_top30은 비용 무차감 PIT 엔진에서 IR 0.07 ≈ 0 — 본 시험의 실질 질문은 "**월별 멀티팩터 선별(grade cut)이 룰 universe에서 엣지를 추가하는가**". 기각 가능성 상당, 등록 기각 누적 7건의 연장선임을 인지하고 시작.
2. fixed18(73% CAGR)과의 직접 경쟁 아님 — 다른 트랙. 비교는 정보용.
3. regime: 18종 기각과 동일 메커니즘 — 회의적 prior 유지. 30종(중형 포함) + 시장 라우팅이 유일한 신규 정보.
4. survivorship: 시장라벨·풀 = 현재 스냅샷 (상장폐지·이전상장 미반영 — 기존 sweep·PIT 한계 동일).
5. 가드레일(시총·ADTV)은 운용 단계 전용 — 백테스트 멤버십은 점수+정적 라벨만 (look-ahead 방지, universe확정 메모 §5-1).
6. grade cut(≥9)은 production 절대 컷 차용 — PIT-proxy 분포에서 picks 수가 월별로 출렁일 수 있음. 컷 튜닝 금지, picks 0 달은 현금. 결과 해석 시 평균 picks 수 병기.

### 해석 (정직)

1. **엔진 동등성 입증**: fixed18 71.43%/IR 1.46은 D §8 장마감 재실행(71.43%)과 동일 — 구조 게이트 882건 = D와 같은 수. 즉 격차는 엔진이 아니라 전략 차이.
2. **비용이 원인 아님**: u_ref(0.235% 참고) 26.49%도 KOSPI에 −3.06%p — 비용 라우팅 drag는 4년 누적 −0.40%p뿐. **룰 universe 자체가 시장(시총가중 KOSPI)을 못 이김** (월 초과수익 음수 24/49). universe확정 메모 §6-3 "pooled_top30 시장초과 ≈ 0" (비용 무차감 PIT 엔진)이 공식 엔진+월별 멀티팩터 선별에서도 재확인 — grade cut이 평균 24.8/30만 걸러 분산 효과 외 엣지 추가 없음.
3. **regime 3차 기각**: MDD 개선(+1.1~+1.2%p)은 또 합격선(+2.0) 미달이고 CAGR·IR 대가 발생 — 18종(D)과 다른 universe·다른 detector 조합에서도 같은 모양. "regime 조정은 한국 reversal 엣지를 갉아먹는다"가 universe 불문 일반화에 가까워짐.
4. **fixed18 71% vs u_base 26%의 격차**가 이 모듈의 가장 큰 정보: 룰 기반 30종 분산 트랙은 fixed-18 트랙(재량 종목선택+고집중)의 수익원을 복제하지 못한다. 06-03 핸드오프 C의 가설(월간 리밸 빈도가 수익원의 상당 부분)은 이 결과와 별개로 유효 — 단 그 검증은 영역 4 몫.
