## 2. 실제 매매룰 백테스트 (FDR 데이터)

```powershell
cd "C:\Users\긍정적인_삶의자세\Desktop\진우퀀트"
python backtest_v372_rules.py
```

**출력**: 5개 시나리오 × {누적·연환산·Sharpe·MDD·IR·turnover·exits}
```
base       {...}              ← v3.7.2 production (월 리밸·동일가중)
+exit      {...}  exits=NN    ← Chandelier 조기청산 추가
+sizing    {...}              ← VKOSPI 익스포저 조절
+full      {...}
+hedged    {...}
```

**볼 것 (합격선, 매매룰 §10)**:
1. `+full` 연환산(net) **≥ `base`** 인가? → 아니면 오버레이가 해(害). 끈다.
2. `+full` MDD **≤ `base`** 인가? (위험 감소가 목적)
3. `turnover` **≤ `base` × 1.5**? (Chandelier가 너무 늘리면 비용 잠식)
4. `+exit` 단독이 `base`보다 나은가? → **한국 reversal 특성상 나쁠 확률 높음**(내 prior).

**정직한 기대**: `base`가 이길 가능성이 충분히 있습니다. 그러면 "오버레이 빼고 v3.7.2 그대로" 가 정답 — 단순함 우선 재확인.

## 3. regime 5요소로 업그레이드 (다음 iteration)

현재 `backtest_v372_rules.py`의 sizing/헤지는 단순 VKOSPI(또는 실현변동성)다.
5요소로 바꾸려면 `run_rules_backtest` 안에서:
```python
import regime_detector_v40 as rg
# 기존: invest = vkospi_invest_frac(vk)
# 교체:
r = rg.detect(panel['_KOSPI'], vkospi=vkospi, picks=list(w),
              sector_map=sectors, dt=d0, prev_state=prev_state,
              vkospi_is_realized=True)
invest = r['invest_frac']; hedge_on = r['hedge_on']; prev_state = r['state']
```
수급(pykrx)·팩터 ON/OFF는 데이터 연결 후 자동 합류(없으면 3요소로 동작).
