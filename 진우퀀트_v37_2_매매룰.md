1. **진입가**: 세션 VWAP 이하 지정가 (추격 매수 금지)

## 3. Layer 2 — Timing (정량화)

### 3.1 진입 가격 — VWAP 이하 지정가
BUY 신호(S+/S/A) 시 세션 VWAP 이하 지정가만. 시장가 추격 금지.
```
현재가 ≤ 세션 VWAP → 시장가 OK
현재가  > 세션 VWAP → 지정가 = VWAP × 1.002
3거래일 미체결 → 신호 폐기
```
백테스트 근사: 일별 `(H+L+2C)/4` typical price.

### 3.2 분할 매수 trigger — v3.7.2 신호 연동
v3.7.2의 **Echo**와 기존 FAR·거래량을 신뢰도 게이트로 사용:

| 조건 | 매수 방식 |
|---|---|
| FAR_BUY + Echo=+1 + 거래량 z≥+2 + 등급 S+/S | **lump-sum** (1회) |
| FAR_BUY + (Echo=+1 또는 z≥+2) | **2분할** (50%×2, 1주 간격) |
| 등급 BUY인데 위 미충족 | **4분할** (25%×4, 2주) |

핵심: 신뢰도 임계 넘으면 분할을 끈다(현금 놀리는 비용 회피, DCA<Lump 근거). Echo=+1이 신뢰 가산 역할.

## 4. Layer 3 — Sizing (정정본: 동일가중-cap + VKOSPI 익스포저)

### 4.1 Base = production 동일가중-cap (불변)
`apply_weight_caps(picks, stock_cap=0.15, sector_cap=0.35)` 그대로.

### 4.2 VKOSPI 레짐 → 총 익스포저(현금비중) 조절 [신규]
종목별 weight를 건드리지 않고, **전체 투입 비율 `invest_frac`**만 조절한다.

```python
if   VKOSPI < 18:  invest_frac = 1.00   # 평온 (옵션: 1.05, 소폭 레버리지 허용)
elif VKOSPI < 25:  invest_frac = 1.00   # 정상
elif VKOSPI < 32:  invest_frac = 0.85   # 경계 — 현금 15%
else:              invest_frac = 0.60   # 공포(≥32) — 현금 40% 버퍼

종목 최종 비중 = base_weight × invest_frac      # 나머지 (1-invest_frac)는 현금
```

**왜 종목별이 아니라 총 익스포저인가**: production이 동일가중이라 종목별 곱은 의미가 약하고, "공포장엔 덜 투입하고 현금 버퍼" 가 진우 님 행동비용(손 떨려서 룰 못 지킴) 해결에 직접적. v3.6 초안의 "신규매수 60% 삭감" 의도를 월 리밸 동일가중 체계에 맞게 일반화한 것.

**percentile 대안**: VKOSPI 대신 KOSPI 실현변동성을 쓸 경우 절대임계(18/25/32)가 스케일이 안 맞으므로 트레일링 1년 분위수(30/70/90 pct)로 대체. → 백테스트에서 둘 다 검증.

**대시보드**: VKOSPI 색상(녹/황/적) + 현재 `invest_frac` 표시.

## 5. Layer 4 — Exit (ATR Chandelier, 월 리밸 사이 조기청산)

### 5.1 기존 −8/−15/−25% 고정 stop 폐기 → Chandelier
```python
ATR14 = Wilder 14일 ATR
peak  = rolling 22일 '종가' high           # 장중 고가 아님
chandelier = peak - 3.0 × ATR14
if close_today < chandelier:  SELL_TRIGGER  # 종가 기준만, 장중 무시
```
경고 단계(모바일): 🟡 종가<peak−2.0×ATR / 🟠 <2.5×ATR(절반 검토) / 🔴 <3.0×ATR(익일 시초가 청산).

### 5.2 월 리밸런스와의 상호작용 (중요)
v3.7.2는 월 1회 등급 기반 리밸. Chandelier는 **리밸 사이**에 작동: 한 달 안에 종가가 청산선을 깨면 다음 리밸을 기다리지 않고 조기 청산 → 현금(또는 헤지)으로. 이게 monthly 선택 위에 얹는 **실행 알파**이자, turnover를 늘리는 요인(백테스트에서 거래비용 차감 후 순효과 확인 필요).

### 5.3 Take-profit 보류
등급 quarterly/월 하락 매도가 실질 take-profit. 단기 익절 룰은 turnover·세금만 늘림.

## 6. Layer 5 — Market Regime / Hedge (신규)

```python
if   KOSPI > MA200*1.02 and VKOSPI < 22:  regime = "RISK_ON"
elif KOSPI < MA200*0.98 and VKOSPI > 28:  regime = "RISK_OFF"
else:                                      regime = "NEUTRAL"
```
**헤지 룰** (RISK_OFF 진입 시):
- NAV의 5~10% KODEX 인버스(1X, 114800) 매수, 보유 ≤4주, NEUTRAL/RISK_ON 전환 시 즉시 청산
- 자금: 현금 우선; 부족 시 base 비중 평균 이상 종목 중 **최저 등급 1개만** trim, **S+/S 불가**
- 금지: 곱버스(2X), 4주 초과, 10% 초과
- regime은 false positive 가능 → 진우 최종 승인(토글)

⚠️ **v4.0 영역 2 regime detector와의 관계**: 이 Layer 5(+Layer 3 VKOSPI)는 v4.0의 한국식 5요소 regime detector(지수추세·VKOSPI 비대칭·수급·섹터집중·팩터 ON/OFF)의 **MVP 선행 버전**이다. v4.0에서 이 둘은 5요소 detector로 흡수·고도화된다. → 지금은 단순 버전으로 매매룰에 넣고, v4.0에서 통합.

**레이어별 결론**:
- Layer 4 Chandelier exit → ❌ 기각
- Layer 3 VKOSPI 익스포저 → ❌ 기각
- Layer 5 헤지 → ❌ 기각 (단 실현변동성이 임계 미달로 헤지 미발동 → 실 KRX VKOSPI로 재검증 여지)
