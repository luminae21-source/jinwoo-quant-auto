# 티어1 활성화 가이드 (2026-06-07)

티어1 = 지금 바로 켜는 운영 안전장치 + 테마 행동규칙. 백테스트·일봉 불필요.
파일: `risk_rules.yaml`(설정) · `tier1_risk_rules.py`(엔진, self-test 14/14 통과).
원칙: production 무수정 — 기존 스크립트는 건드리지 않고 신규 파일만 호출.

## 1. 본체 MDD 메타룰 — 매월 1회
`live_tracker.py`로 실현수익 기록 직후 상태만 확인:
```bash
python live_tracker.py --add 2026-06 4.2 1.1   # 기존 워크플로 (port%, kospi%)
python tier1_risk_rules.py --check-mdd          # live_record.csv 읽어 판정
```
- **정상**: 그대로 진행.
- **경보**(최근월 ≤ −10%): 다음 리밸런스서 원인 점검.
- **축소**(누적낙폭 ≤ −18%): 노출 축소 + 전수 점검. ← 백테스트 MDD −12.2%의 1.5배 초과 신호.

## 2. 테마 진입 — 매수 전 1회
매수 직전 후보·현재 lane 상태를 넣어 게이트 통과 여부 확인:
```bash
python tier1_risk_rules.py --demo               # 입력 형식 예시
```
코드에서 호출 시 `check_theme_entry(cfg, lane_state, candidate)` →
`ALLOW / HOLD(급등 보류) / BLOCK(킬스위치·VI·집중한도·하한가)`.
검사 항목: 월/분기 킬스위치 · VI·상한가 · 직전봉 +10% 급등 · 손절폭 −20% 초과 ·
동일테마 2종목 · 1종목 5% · lane 20% · 보유 5종목.

## 3. 분할·익절 규율 (수동 체크리스트)
정찰 1/3 → 추세·거래량 확인 후 잔여 / 목표 도달 시 1/2 익절(잔량 트레일링은 티어2).

## 4. 지금 결정할 것 하나 — 바이오 3→2 축소
워치리스트 바이오가 3종목이라 "동일테마 ≤2" 규칙 위반. **2종목으로 줄이거나 합산비중을 lane 40% 안에 맞춰야** 함. 객관 데이터(`kosdaq_theme_watchlist_진우기입.csv`):

| 종목 | 시총(억) | ADTV(억) | 부채비율 |
|---|---|---|---|
| 알테오젠 | 191,303 | 1,880 | 0.54 |
| HLB | 69,396 | 431 | 0.88 |
| 펩트론 | 63,073 | 547 | 0.26 |

→ 어느 2종목을 남길지는 **진우의 thesis 판단**(가드레일=고려가능이지 매수신호 아님). 결정 후 `risk_rules.yaml`은 그대로 두고 워치리스트에서 1종목 제외하면 됨.
