### 🔴 영역 4 — 종목 교체 룰 + 변동 Attribution (가장 어려움)

**새 대화에 붙여넣을 prompt:**

```

문제 정의:
종목 X가 ±10% 변동했을 때, 그 이유가:
(a) 퀀트 펀더멘털 변화 — F-Score 등급 변동, 분기 실적 발표
(b) 모멘텀 — 1M/12M 추세, 거래량 패턴
(c) 산업 로테이션 — 같은 산업 동반 등락
(d) 매크로 — 금리, 환율, 외인 수급
(e) 이벤트 — M&A, 분할, 자사주 매입, 배당
(f) 시장 전체 — KOSPI 동조 움직임

이걸 구분 못 하면 교체 룰이 단순 감(感)이 됨.
"왜 빠졌는지" 모르면 "팔지 말지" 판단 불가능.

리서치 필요 (논문 학습 — 우선순위 순):
1. Fama & French 3-factor (1993): Size, Value, Market
2. Carhart 4-factor (1997): + Momentum
3. Fama & French 5-factor (2015): + Profitability, Investment
4. Asness, Moskowitz, Pedersen (2013) — "Value and Momentum Everywhere"
5. Daniel-Kent-Wermers (1997) — Characteristic-based benchmark
6. Kim & Park 한국 시장 factor (2010~)
7. Hou, Xue, Zhang (2015) — q-factor model
8. Idiosyncratic vs Systematic return decomposition
9. Event study methodology (MacKinlay 1997)
10. Stambaugh, Yuan (2017) — Mispricing factor

목표 (Phase별):
Phase 1: 시장 전체 vs 종목 분해 (CAPM β)
Phase 2: 산업 factor 추가 (산업 ETF 수익률 회귀)
Phase 3: F·Mom·Quality 등 quant factor 추가
Phase 4: 이벤트 detection (실적·공시·뉴스 충격)

기술 스택:
- Python: statsmodels (OLS), scikit-learn
- 데이터: FDR (5년치 일별 수익률 18~50종목 + KOSPI + 산업 ETF)
- 시각화: 대시보드 attribution 패널 추가

질문 (새 대화에서):
1. 한국 시장에 가장 잘 맞는 factor model은? (FF3? FF5? Carhart?)
2. 18~50종목 규모에서 factor 분석 통계적 유의성 확보 방법?
3. 진우가 매일 모바일 대시보드에서 볼 수 있는 형태로 attribution 표시?
   - "삼성전자 +5% = 시장 +1% / 반도체 +3% / 종목 고유 +1%"
4. 교체 trigger 정량 룰:
   - 펀더멘털 악화 (F-Score 2등급 하락) → 즉시 교체
   - 모멘텀 일시 약화 → 보유
   - 산업 로테이션 → 산업 비중 조정만, 종목은 유지
   - Idiosyncratic 충격 (이벤트) → 케이스별 판단
5. False signal 제거 (잡음과 신호 구분 — z-score 기준?)

예상 기간: 4~6주
선행 작업: 영역 1·2·3 완료 (퀀트 기법·매매룰·universe 확정)

참조 파일:
- 진우퀀트_v37_universe.md (영역 3 결과)

```

## 6. 리스크·주의사항

### 영역 1 리스크
- 다중공선성 (factor 간 상관관계 높으면 alpha 사라짐)
- Overfitting (백테스트 좋아도 실제 의미 없음)
- **해결:** 단순함 우선, factor 추가 시 신중

### 영역 2 리스크
- 매매 빈도 증가 → 수수료·세금 누적
- 진우 라이프스타일 위배 → 룰 따라가기 어려움
- **해결:** 정량화 + 진우 실행 가능성 검증

### 영역 3 리스크
- universe 확장 = 모니터링 부담 증가
- 소형주 = 유동성·정보 비대칭 리스크
- **해결:** 자동화 보조 + 진우의 한계 인정

### 영역 4 리스크
- Factor model 통계 검정력 부족 (N=18~50 작음)
- 학습 곡선 가장 가파름
- 결과 해석 오류 가능성
- **해결:** 단계적 접근 (Phase 1부터), 외부 검증자 도움
