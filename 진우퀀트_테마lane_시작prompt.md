## 2. 접근 방식 — forward 신호를 어떻게 잡나
backward(과거재무)가 구조적으로 못 보는 것을 forward로:
- **신호 후보**: 카탈리스트(수주·신제품·정책·CAPEX)·산업 사이클·수급 쏠림 — *과거 재무에 안 잡히는* 선행 정보
- **데이터 소스**: (현재) 로컬 산업분류·가격·유동성으로 충분 / (선택 확장) 외국인·기관 수급, 테마 ETF 자금, 밸류
- **파이프라인**(발굴_시스템 5단계): 테마 식별(top-down) → value-chain 매핑 → 가드레일 → **선반영(crowding) 체크** → thesis→실행→측정

## 3. ⚠️ 규율 장치 — "테마 추격"으로 변질 방지 (이 lane의 핵심)
재량은 overfitting·hindsight·뇌동매매 위험이 큼. **그래서 사후 검증 가능한 형태로 묶는다:**
- **(가) thesis + 무효화 트리거 사전등록** — 사기 *전에* "이러면 철회/손절" 적기 → 사후 합리화 차단
- **(나) 선반영 rubric** — 급등(+100%↑)·고점근접·보편화 뉴스 = late → 패스. **카탈리스트가 *남았는지*만** 본다
- **(다) Track W 반사실 측정** — 재량 픽 vs *같은 돈으로 시스템 픽 들었을 때*. **이게 forward 합격선의 본체**(절대수익 아님)
- **(라) rule-ify** — 거를 수 있는 건 룰(가드레일=생존), 못 거르는 건 진우(thesis 1개), 판정은 데이터(반사실). "느낌"의 자리는 thesis 하나뿐
- **(마) cap·격리** — 틀려도 안 죽게(≤10%·1~2슬롯)
- **연결**: B전환 **W(위성) 트랙 = 이 lane** (대덕전자 첫 실증, task #18). 신규 트랙 아님. 시스템 트랙·KOSDAQ 체계배분 0 불변.
